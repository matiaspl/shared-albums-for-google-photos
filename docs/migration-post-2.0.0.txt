TITLE: Important: Breaking changes in version 2.0.0 — Migration guide

BODY:

Hi everyone,

Version 2.0.0 introduces new features like experimental video support and, based on user feedback, makes 'gallery' the default mode. I also took the liberty to clean up naming inconsistencies in shortcode parameter names, default values, and allowed values while the plugin is still young and the user base is small. <strong>This is a one-time cleanup — from 2.0.0 onward, all parameter names and values will remain stable and backward-compatible.</strong>

If you are upgrading from 1.0.x, you will need to update your shortcodes. Below is a complete migration guide.

<strong>Renamed Parameters</strong>

<code>autoplay</code> → <code>slideshow</code>
<code>autoplay-delay</code> → <code>slideshow-delay</code>
<code>full-screen-autoplay</code> → <code>fullscreen-slideshow</code>
<code>full-screen-autoplay-delay</code> → <code>fullscreen-slideshow-delay</code>
<code>autoplay-inactivity-timeout</code> → <code>slideshow-autoresume</code>
<code>slideshow-inactivity-timeout</code> → <code>slideshow-autoresume</code>
<code>full-screen-switch</code> → <code>fullscreen-toggle</code>
<code>image-width</code> → <code>source-width</code>
<code>image-height</code> → <code>source-height</code>
<code>max-photos-per-album</code> → <code>limit</code>

<strong>Changed Default Values</strong>

<code>width</code>: 267 → 400 (better default container size)
<code>height</code>: 200 → 300 (better default container size)
<code>slideshow</code>: true → false (autoplay should be opt-in)
<code>slideshow-delay</code>: "4-12" (random) → "5" (predictable behavior)
<code>fullscreen-slideshow</code>: true → false (autoplay should be opt-in)
<code>fullscreen-slideshow-delay</code>: "3" → "5" (consistent with inline delay)
<code>source-width</code>: 1920 → 800 (faster loading; set source-width="1920" for high-res)
<code>source-height</code>: 1440 → 600 (faster loading; set source-height="1440" for high-res)
<code>fullscreen-toggle</code>: "double-click" → "button-only" (more discoverable for new users)
<code>background-color</code>: "#FFFFFF" → "transparent" (blends with your page theme)
<code>start-at</code>: "random" → "1" (predictable behavior)
<code>mode</code>: "single" → "gallery" (gallery is now the default experience)

<strong>Changed Values</strong>

<code>mode</code>: "single" has been renamed to "slider". "carousel-to-single" has been removed. "gallery" is a new mode.
<code>image-fit</code>: "stretch" has been removed (use "cover" instead).
<code>fullscreen-toggle</code>: "single-click" has been renamed to "click".

<strong>Removed Parameters</strong>

<code>full-screen-navigation</code>: This parameter has been removed. Fullscreen navigation is now controlled via <code>fullscreen-toggle</code> (set to "disabled" to prevent entering fullscreen entirely) and <code>interaction-lock</code> (set to "true" to disable all user interaction).

<strong>New Parameters in 2.0.0</strong>

The following new parameters have been added:

<ul>
<li><code>fullscreen-source-width</code>, <code>fullscreen-source-height</code> — separate source resolution for fullscreen mode (default 1920x1440)</li>
<li><code>fullscreen-image-fit</code> — image fit mode for fullscreen, independent from inline (default "contain")</li>
<li><code>controls-color</code> — color of slider/gallery UI controls</li>
<li><code>video-controls-color</code> — color of video player controls</li>
<li><code>show-navigation</code> — show/hide prev/next arrows (default true)</li>
<li><code>show-videos</code> — include or exclude video entries (default false)</li>
<li><code>video-controls-autohide</code> — auto-hide video controls (default false)</li>
<li><code>interaction-lock</code> — lock all user interaction, useful for digital signage (default false)</li>
<li><code>gallery-layout</code> — gallery thumbnail layout: "grid" or "justified" (default "grid")</li>
<li><code>gallery-sizing</code> — gallery thumbnail sizing: "ratio" or "fill" (default "ratio")</li>
<li><code>gallery-columns</code> — number of gallery columns on desktop (default 3)</li>
<li><code>gallery-columns-tablet</code> — number of gallery columns on tablet (default 2)</li>
<li><code>gallery-columns-mobile</code> — number of gallery columns on mobile (default 1)</li>
<li><code>gallery-row-height</code> — row height in pixels for justified layout (default 200)</li>
<li><code>gallery-rows</code> — number of gallery rows per page, 0 = show all (default 0)</li>
<li><code>gallery-scrollable</code> — enable scrollable gallery container (default false)</li>
</ul>

<strong>Quick migration example</strong>

Before (1.0.x):
<code>[jzsa-album link="..." mode="single" autoplay="true" autoplay-delay="5" full-screen-switch="single-click" image-width="1920" image-height="1440"]</code>

After (2.0.0):
<code>[jzsa-album link="..." mode="slider" slideshow="true" slideshow-delay="5" fullscreen-toggle="click" source-width="1920" source-height="1440"]</code>

For the full parameter reference with live examples, visit the plugin's <strong>Settings</strong> page in your WordPress admin (Settings → Shared Albums for Google Photos).

I apologize for any inconvenience. These changes were necessary to establish clear, consistent, and industry-standard naming before the plugin grows further. Going forward, backward compatibility will be maintained in all future releases.
